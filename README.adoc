= PipeWatch

A macOS/Windows menu bar application that monitors delivery pipeline status.
Currently just for GitLab CI.

== Features

* System tray icon showing pipeline status (green/red/yellow/gray)
* Quick access menu with pipeline details
* Configurable refresh interval

== Installation

Download the latest release from https://github.com/Richargh/pipewatch/releases[GitHub Releases].

Releases bundle the JDK, so no separate Java installation is required. This is provided by https://conveyor.hydraulic.dev/[Conveyor] (see <<Releasing>>).

=== Windows

* `pipewatch.exe` — Recommended installer (installs certificate + app)
* `pipewatch-<version>.x64.msix` — MSIX package (requires certificate installed first)
* `pipewatch-<version>-windows-amd64.zip` — Portable ZIP

=== macOS

* `pipewatch-<version>-mac-aarch64.zip` — Apple Silicon (M1-Mx)
* `pipewatch-<version>-mac-amd64.zip` — Intel Macs

=== Linux

* `pipewatch_<version>_amd64.deb` — Debian/Ubuntu package
* `pipewatch-<version>-linux-amd64.tar.gz` — Tarball

NOTE: The release also contains files for auto-updates and self-signed certificate installation (`.appinstaller`, `.crt`, `.rss`, `Packages`, etc.). These are used for auto-updates and don't need to be downloaded manually.

== Requirements

* JDK 21+
* GitLab Personal Access Token (PAT) with `read_api` scope

=== Setup with SDKMAN

[source,bash]
----
sdk env install
----

This installs (if needed) and activates the Java version from `.sdkmanrc`.

=== Creating a GitLab Personal Access Token

To monitor your pipelines, you need a GitLab Personal Access Token with read access:

1. Log in to your GitLab instance
2. Click your avatar (top-right corner) → *Edit profile*
3. In the left sidebar, select *Access Tokens*
4. Click *Add new token*
5. Configure your token:
* *Token name*: e.g., "Pipeline Monitor"
* *Expiration date*: Set an appropriate expiration (max 365 days)
* *Scopes*: Select `read_api` (read-only access to the API)
6. Click *Create personal access token*
7. *Important*: Copy and save your token immediately — you won't be able to see it again

TIP: For security, only grant the minimum required scope (`read_api`). Never share your token or commit it to version control.

== Usage

=== Run (Development)

[source,bash]
----
./gradlew run
----

=== Test

[source,bash]
----
./gradlew test
----

=== Lint

[source,bash]
----
# Check for issues
./gradlew ktlintCheck

# Auto-fix issues
./gradlew ktlintFormat
----

=== Build Distribution (Local)

[source,bash]
----
# macOS DMG
./gradlew packageDmg

# Windows MSI
./gradlew packageMsi

# Current platform
./gradlew packageDistributionForCurrentOS
----

== Releasing

Releases are built with https://conveyor.hydraulic.dev/[Conveyor] and published to GitHub Releases.

1. Update `CHANGELOG.md` with the new version
2. Commit and push
3. Create and push a tag: `git tag v0.2.0 && git push origin v0.2.0`
4. Go to Actions → *release* workflow → *Run workflow*
5. Enter the tag (e.g., `v0.2.0`) and run

The workflow builds installers for all platforms (macOS, Windows, Linux) and creates a GitHub release with notes from the changelog.
